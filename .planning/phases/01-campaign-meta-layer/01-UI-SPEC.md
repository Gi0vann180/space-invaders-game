## Interaction Contract

This phase must enforce the following interaction states, aligned to locked decisions in context:

1. Campaign map state model
- Show phases 1-5 in linear connected nodes.
- Node states: locked, unlocked, completed.
- Locked nodes remain visible with lock indicator and unlock hint.

2. Completion and unlock flow
- Completion trigger is boss defeat for current phase.
- Show completion overlay with title "FASE COMPLETADA", reward summary, and next action.
- Unlock next phase automatically after completion.

3. Retry and replay behavior
- Defeat state presents immediate retry action for same phase.
- Completed phases remain replayable without losing permanent progression.

4. Pause/resume and reopen flow
- On pause/background and app reopen with interrupted run, present binary choice: "Continuar" or "Reiniciar".
- Interrupted run does not expire in v1 while snapshot is valid.
- If snapshot invalid/corrupted, recover to start of current phase and notify player with a non-blocking warning.

5. Offline behavior
- No internet dependency for campaign map, unlock visualization, or continue/restart flow.
- Any network-dependent UI elements are hidden in this phase.
