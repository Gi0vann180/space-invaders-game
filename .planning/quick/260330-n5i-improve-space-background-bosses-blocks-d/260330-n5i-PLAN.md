  <behavior>
    - Test: evaluateStageProgression(bossStage, 0 enemies, bossActive=false, bossDefeated=true) returns enterShop=true, enterBossFight=false
    - Test: evaluateStageProgression(bossStage, 0 enemies, bossActive=false, bossDefeated=false) returns enterBossFight=true (normal spawn)
    - Test: evaluateStageProgression(bossStage, 0 enemies, bossActive=true, bossDefeated=false) returns enterShop=false, enterBossFight=false (fight in progress)
    - Test: evaluateStageProgression(bossStage, 0 enemies, bossActive=true, bossDefeated=true) returns enterShop=true (boss just died, still referenced)
  </behavior>
  <action>
    ROOT CAUSE: In `evaluateStageProgression`, the `if (!bossActive)` check on line 40 runs BEFORE the `if (!bossDefeated)` check on line 49. When the collision system defeats a boss, it sets `wave.boss = null`, so `bossActive` becomes `false`. The function then hits the `!bossActive` branch and returns `enterBossFight: true`, re-spawning the boss endlessly.

    FIX: Reorder the logic in the boss stage branch so that `bossDefeated` is checked FIRST:

    ```
    if (bossStage) {
      if (bossDefeated) {
        // Boss was just killed — advance to shop
        return { enterShop: true, enterBossFight: false, nextStage: currentStage + 1, ... }
      }
      if (!bossActive) {
        // No boss yet and not defeated — spawn boss
        return { enterShop: false, enterBossFight: true, nextStage: currentStage, ... }
      }
      // Boss fight in progress
      return { enterShop: false, enterBossFight: false, nextStage: currentStage, ... }
    }
    ```

    Create test file `frontend/tests/unit/quick-boss-respawn-after-defeat.test.ts` with the behavior cases above. Use stage 10 (a boss stage per `isBossStage`). Write tests first (RED), then fix the production code (GREEN).

    Code style: no semicolons, single quotes, no trailing commas.
