# Requirements: Space Invaders Mobile

**Defined:** 2026-03-23
**Core Value:** Entregar uma sessao arcade curta, divertida e clara no celular, em que o jogador aprende jogando e sente progressao real sem perder a simplicidade do Space Invaders.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Controls & UX

- [ ] **CTRL-01**: User can control the ship through touch input with responsive and comfortable handling on Android phones
- [ ] **CTRL-02**: User can adjust comfort options including sensitivity, touch target behavior, and motion intensity reduction
- [ ] **CTRL-03**: User can restart a run and return to gameplay with low friction after defeat or interruption

### Combat & Clarity

- [ ] **COMB-01**: User can clearly distinguish projectiles, hazards, pickups, and impact feedback on small mobile screens
- [ ] **COMB-02**: User learns core movement, risk, reward, and loop behavior through early gameplay without intrusive tutorial flow
- [ ] **COMB-03**: User receives clear end-of-run feedback about outcome, rewards, and progress

### Progression

- [ ] **PROG-01**: User can understand what was unlocked, why it matters, and what the next progression target is
- [ ] **PROG-02**: User encounters enough early and mid-game novelty to reduce repetitive run perception

### Android Release

- [ ] **ANDR-01**: User session survives pause, resume, and common Android interruptions with reliable continuity
- [ ] **ANDR-02**: Team can generate a publishable Android package from the existing codebase without duplicating game logic
- [ ] **ANDR-03**: User experiences stable performance and reliability on target Android devices during real play sessions

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Advanced Progression

- **PROG-03**: User unlocks meaningful content through discovery and play patterns, not only currency spend
- **PROG-04**: User can pursue mastery objectives tied to skill and technique
- **PROG-05**: User can choose build identities with clear gameplay tradeoffs

### Release Operations

- **RELS-01**: Team has full Play Store operations assets and release checklist beyond the baseline Android packaging flow

## Out of Scope

Explicitly excluded to avoid scope creep in this milestone.

| Feature | Reason |
| ------- | ------ |
| Complex online backend with accounts and proprietary infrastructure | High cost and complexity; not required to validate the core mobile loop |
| Multiplayer | Outside current solo and low-cost scope; not required for first release value |
| Simultaneous Android and iOS launch | Android-first sequencing reduces operational and technical risk |
